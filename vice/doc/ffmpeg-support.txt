1 = broken in 3.1, fixed in trunk
2 = using --external-ffmpeg will be even worse, that requires building ffmpeg
    from source and installing it into an alternate prefix so it won't bork
    the ffmpeg lib used by the Webpositive browser
3 = VICE's code needs to have CPPFLAGS to include "-I/usr/local/include" and
    LDFLAGS to include "-L/usr/local/lib" to make it find <png.h>, while using
    these flags will break the static ffmpeg compile (OpenBSD will use its own
    ffmpeg headers and libs, resulting in a ton of deprecation and other
    messages and finally borking with an "expected ;"-like message).
    When NOT using --enable-static-ffmpeg, altering the CPPFLAGS/LDFLAGS isn't
    required and VICE builds fine, so somehow the static ffmpeg compile breaks
    these flags.
